# Changelog

## v0.3.0

### 新增

- 全新更新對話框：直接顯示 release notes，可一鍵在 app 內下載並靜默安裝，整個過程不再離開 app。
- 設定頁新增「立即檢查更新」按鈕，可手動觸發檢查並顯示結果。
- 更新對話框可選擇「跳過此版本」，下次啟動不再提示；設定頁可清除已跳過的版本。

### 調整

- GUI 框架從 customtkinter 遷移至 PySide6，外觀改為原生深色主題，HiDPI 支援更完善。
- 更新對話框的 release notes 改用 Qt 原生 Markdown 渲染，支援完整語法（表格、圖片等）。
- 安裝程式改為支援靜默升級（/VERYSILENT），自動關閉舊版並啟動新版，使用者不再看到精靈式安裝畫面（升級到此版本後生效）。
- 版本號解析改用嚴格 regex，遇到無法解析的 tag（如預發版 `-rc1`）會直接視為「無更新」而非崩潰。

### 破壞性變更

- GUI 框架從 customtkinter 換為 PySide6。開發環境需重新執行 `uv sync` 安裝新依賴。對終端使用者無影響（installer 不變）。

### 修正

- 16:10、21:9 等非 16:9 視窗模板縮放比例錯誤，導致比對失準。改為以「16:9 參考框塞入視窗」的最大縮放比（寬高比例取較小者）計算。
- 使用者覆寫的模板會被當成 bundled 模板再縮放一次，導致比對失敗。改為自製模板一律以 ratio=1.0 載入（視為已在當前解析度下截圖），只有 bundled 模板會套用縮放。

## v0.2.0

### 新增

- 所有步驟顯示「耗時」統計，方便掌握每次執行花費的時間。

### 調整

- 根據使用者回饋，將預設 `threshold` 從 0.90 降為 0.85、`between_press_ms` 從 200 提高為 250，整體運行更穩定。

### 修正

- Windows 使用者名稱含非 ASCII 字元（如中文）時無法載入模板的問題。

### 破壞性變更

- 設定檔中 `[delete]` section 已重新命名為 `[remove_car]`。升級後舊設定會被忽略並使用預設值，下次儲存時會自動寫入新格式。如需保留原設定，請手動將 `config.ini` 中的 `[delete]` 改為 `[remove_car]`。

## v0.1.0

### 新增

- 功能：刷技能點、購買車輛、升級車輛、移除車輛
- GUI 設定頁面
- 啟動時檢查更新
